## 策略概述

### 1.1 策略类型
BG自动分析交易系统采用**多因子量化策略**，结合技术分析、市场情绪和风险控制，通过五维评分系统生成交易信号。

### 1.2 核心理念
- **客观分析**：基于定量指标而非主观判断
- **多维度验证**：五个独立维度交叉验证，降低误判概率
- **动态适应**：根据市场条件调整参数和策略
- **风险第一**：在风险可控前提下追求收益

### 1.3 适用市场环境
- **震荡市**：通过震荡策略获取波段收益
- **趋势市**：跟随趋势放大收益
- **突破市**：识别关键点位突破机会
- **反转市**：识别顶部底部反转信号

## 市场分析逻辑

### 2.1 数据预处理
#### 数据清洗
- **异常值检测**：使用Z-score和IQR方法检测异常K线
- **数据对齐**：确保多时间周期数据时间戳对齐
- **缺失值处理**：插值或标记缺失数据
- **数据验证**：验证OHLCV数据的合理性

#### K线有效性验证
```python
def validate_kline(kline):
    # 检查价格关系
    assert kline.high >= kline.low, "High must be >= Low"
    assert kline.high >= kline.open >= kline.low, "Open must be within high-low range"
    assert kline.high >= kline.close >= kline.low, "Close must be within high-low range"
    # 检查成交量非负
    assert kline.volume >= 0, "Volume must be non-negative"
```

### 2.2 技术指标计算
#### 核心指标类别
- **趋势指标**：EMA、MA、MACD、DMI
- **振荡指标**：RSI、KD、WR
- **波动率指标**：ATR、Bollinger Bands、标准差
- **成交量指标**：成交量移动平均、成交量变异率

#### 指标计算优先级
1. **基础指标**：MA、EMA、ATR
2. **衍生指标**：MACD、RSI、Bollinger Bands
3. **复合指标**：趋势强度、波动率评分

## 五维评分系统

### 3.1 评分维度定义

#### 维度一：成交量分析 (Volume Score, 15%权重)
**评分逻辑**：
- 识别成交量放大或萎缩
- 判断市场参与度
- 验证价格变动的有效性

**计算方法**：
```
volume_sma_ratio_score = (当前成交量 / 20日成交量SMA) * 权重1
volume_spike_score = 最近5日成交量峰值 / 历史最大成交量 * 权重2
volume_trend_score = 10日成交量趋势 * 权重3
volume_score = volume_sma_ratio_score + volume_spike_score + volume_trend_score
```

**应用场景**：
- 成交量放大 + 价格上涨 = 强势信号
- 成交量萎缩 + 价格高位 = 反转信号
- 成交量放大 + 价格下跌 = 抛售信号

#### 维度二：K线形态分析 (Candlestick Score, 15%权重)
**评分逻辑**：
- 识别经典K线形态
- 评估形态的可靠性和意义
- 结合市场背景判断形态有效性

**主要形态识别**：
- **反转形态**：锤子线、射击之星、吞没形态、孕线
- **持续形态**：上升三法、下降三法、十字星
- **强度形态**：长上影、长下影、光头光脚

**计算方法**：
```
pattern_weight = 形态重要性 * 形态可靠性 * 市场背景匹配度
candlestick_score = Σ(识别到的形态权重) + 形态排列评分 + 价格位置评分
```

**评分细则**：
- 锤子线(底部)：+10分
- 射击之星(顶部)：+8分
- 看涨吞没：+12分
- 看跌吞没：+12分
- 十字星：+3分

#### 维度三：技术指标分析 (Technical Score, 25%权重)
**评分逻辑**：
- 综合多个技术指标判断
- 识别指标背离和共振
- 评估技术面强度

**核心指标组合**：
- **RSI超买超卖**：RSI > 70或 < 30
- **MACD趋势**：MACD线与信号线交叉
- **均线多空排列**：5/10/20日均线相对位置
- **布林带突破**：价格突破布林带上轨或下轨

**计算方法**：
```
rsi_score = f(RSI当前值, RSI趋势)
macd_score = f(MACD金叉死叉, MACD柱状图变化)
ma_alignment_score = f(均线排列状态)
bb_breakout_score = f(布林带突破位置)
technical_score = (rsi_score * rsi_weight) + (macd_score * macd_weight) + ...
```

#### 维度四：趋势强度分析 (Trend Strength Score, 25%权重)
**评分逻辑**：
- 评估当前趋势的强度和可持续性
- 识别趋势转折点
- 判断趋势的内在动力

**分析要素**：
- **EMA多空关系**：EMA12与EMA26相对位置
- **价格与均线关系**：价格在均线的相对位置
- **均线排列一致性**：多条均线的排列状态
- **动量指标**：价格变化的加速度

**计算方法**：
```
ema_trend_score = f(EMA快慢线关系, EMA趋势持续性) * 权重
price_position_score = f(价格相对均线位置, 位置合理性) * 权重
moving_avg_alignment_score = f(均线排列状态, 排列一致性) * 权重
momentum_score = f(价格动量, 动量变化率) * 权重
trend_strength_score = ema_trend_score + ... + momentum_score
```

#### 维度五：波动率分析 (Volatility Score, 20%权重)
**评分逻辑**：
- 评估市场波动特征
- 识别高波动和低波动环境
- 判断波动率的持续性

**分析指标**：
- **ATR百分比**：ATR相对于价格的百分比
- **布林带宽度**：布林带上下轨的相对距离
- **历史波动率**：基于收盘价的标准差

**计算方法**：
```
atr_percent_score = f(ATR百分比的合理性, 波动率水平)
bb_width_score = f(布林带宽度, 布林带扩张收缩状态)
historical_vol_score = f(历史波动率, 波动率趋势)
volatility_score = (atr_percent_score * atr_weight) + ...
```

### 3.2 权重配置策略
#### 静态权重
```
WEIGHTS = {
    'volume': 0.15,
    'candlestick': 0.15,
    'technical': 0.25,
    'trend_strength': 0.25,
    'volatility': 0.20
}
```

#### 动态权重调整
根据市场状态调整各维度权重：
- **趋势市**：增加趋势强度权重
- **震荡市**：增加技术指标权重
- **高波动**：增加波动率权重
- **低波动**：增加成交量权重

## 信号生成机制

### 4.1 信号类型定义
#### 买入信号 (Buy Signal)
**触发条件**：
- 综合评分 > 70分
- 不存在明显卖出信号
- 风险指标在可控范围内
- 资金充足度 > 预设阈值

#### 卖出信号 (Sell Signal)
**触发条件**：
- 综合评分 < 30分
- 不存在明显买入信号
- 持仓浮盈达到止盈条件
- 风险指标超过警戒线

#### 持有信号 (Hold Signal)
**触发条件**：
- 30 ≤ 综合评分 ≤ 70
- 无明确买入或卖出信号
- 持仓状态维持

### 4.2 信号强度分级
#### 强信号 (Strong Signal)
- 评分 > 80 或 < 20
- 多个维度同向确认
- 技术指标高度共振
- 成交量明显配合

#### 中等信号 (Medium Signal)
- 评分 70-80 或 20-30
- 部分维度确认
- 技术指标基本支持
- 成交量一般配合

#### 弱信号 (Weak Signal)
- 评分 60-70 或 30-40
- 单一维度支持
- 技术指标分歧
- 成交量配合度低

### 4.3 信号确认机制
#### 时间确认
- 信号出现后等待1-2个K线确认
- 避免假突破带来的误判
- 确保信号的稳定性

#### 交叉验证
- 多时间周期信号验证
- 不同指标相互验证
- 基本面因素验证

## 交易执行策略

### 5.1 订单类型选择
#### 市价单 (Market Order)
**适用场景**：
- 信号强度高的交易
- 追踪突破行情
- 交易执行时间敏感

**风险控制**：
- 设置滑点保护
- 控制交易规模
- 监控市场流动性

#### 限价单 (Limit Order)
**适用场景**：
- 信号强度中等的交易
- 等待回调机会
- 交易执行时间不敏感

**价格设定**：
- 买入价格：当前价格 * (1 - 限价偏差)
- 卖出价格：当前价格 * (1 + 限价偏差)

### 5.2 仓位管理策略
#### 金字塔加仓
**规则**：
- 首次建仓：20%资金
- 第一次加仓：15%资金（价格有利变动后）
- 第二次加仓：10%资金（趋势确认后）
- 总仓位不超过50%

#### 逆金字塔减仓
**规则**：
- 部分止盈：盈利30%时减仓30%
- 止盈加码：盈利50%时减仓50%
- 动态止盈：根据波动率调整止盈位

### 5.3 执行时序策略
#### S级（紧急）：信号强度 > 90
- 立即执行
- 100%资金执行
- 无等待确认

#### A级（重要）：信号强度 80-90
- 快速执行
- 75%资金执行
- 简单确认后执行

#### B级（一般）：信号强度 70-80
- 标准执行
- 50%资金执行
- 需要多重确认

#### C级（次要）：信号强度 60-70
- 慎重执行
- 25%资金执行
- 充分确认后执行

## 风险管理策略

### 6.1 止损机制
#### 技术止损
- **固定比例止损**：亏损超过持仓价值3%
- **技术位止损**：跌破关键技术位
- **ATR止损**：基于平均真实波幅设定

#### 时间止损
- **持仓时间限制**：单笔持仓不超过X天
- **信号失效**：信号生成后Y小时内未触发
- **市场环境恶化**：市场转为不利状态

### 6.2 资金管理
#### 单笔风险限额
- 单笔交易风险不超过总资金的1%
- 最大亏损控制在总资产的5%以内

#### 组合风险管理
- 相关性控制：避免同向持仓过度集中
- 时间分散：避免所有交易同时进行
- 品种分散：多品种对冲风险

### 6.3 系统风险控制
#### 连续亏损控制
- 连续亏损3笔后暂停策略1小时
- 连续亏损5笔后降低仓位规模
- 连续亏损8笔后全面暂停策略

#### 日亏损限额
- 单日亏损超过总资产2%时暂停交易
- 连续两日亏损超过1%时降低仓位

## 参数优化与回测

### 7.1 参数分类
#### 不变参数
- 数据采样频率
- 核心指标定义
- 基础权重结构

#### 可调参数
- 指标计算周期
- 评分阈值
- 权重重配比例

#### 动态参数
- 风险阈值
- 仓位比例
- 止盈止损位

### 7.2 优化方法
#### 网格搜索
- 在预设参数范围内进行穷举搜索
- 评估参数组合的表现

#### 遗传算法
- 通过模拟进化过程寻找最优参数
- 适用于多维参数空间优化

#### 贝叶斯优化
- 基于概率模型的序列化参数优化
- 适用于计算成本较高的优化

### 7.3 回测指标
#### 收益指标
- **总收益率**：期末净值 / 期初净值
- **年化收益率**：总收益率 / 年数
- **超额收益率**：相对于基准的收益率

#### 风险指标
- **最大回撤**：从最高点到最低点的最大跌幅
- **波动率**：收益率的标准差
- **下行波动率**：负收益率的标准差

#### 风险调整收益指标
- **夏普比率**：(年化收益率 - 无风险利率) / 年化波动率
- **索提诺比率**：(年化收益率 - 目标收益率) / 下行波动率
- **Calmar比率**：年化收益率 / 最大回撤

## 多时间周期融合

### 8.1 时间周期定义
#### 超短周期 (1M, 5M)
- **用途**：精确进场出场
- **权重**：10%
- **策略**：日内高频交易

#### 短周期 (15M, 30M)
- **用途**：短期趋势判断
- **权重**：20%
- **策略**：波段操作

#### 中周期 (1H, 2H)
- **用途**：中期趋势判断
- **权重**：40%
- **策略**：核心交易决策

#### 长周期 (4H, 日线)
- **用途**：大趋势判断
- **权重**：30%
- **策略**：仓位管理和风险控制

### 8.2 周期融合逻辑
#### 顺势融合
- 多周期同向时增强信号强度
- 低周期信号服从高周期方向

#### 背离检测
- 多周期方向相反时降低交易频率
- 高周期反转时清空相关持仓

#### 延迟确认
- 低周期信号需高周期确认
- 避免低周期噪音干扰

## 策略验证方法

### 9.1 历史回测
#### 数据样本
- **训练集**：70%历史数据
- **验证集**：15%历史数据
- **测试集**：15%历史数据

#### 回测周期
- **最小回测周期**：1年
- **推荐回测周期**：3-5年
- **牛市+熊市**：确保覆盖不同市场环境

### 9.2 前瞻偏差检验
#### 数据泄露检查
- 确保未来数据不被用于当前决策
- 验证实时数据处理逻辑

#### 参数优化检验
- 避免过拟合历史数据
- 在样本外数据上验证参数有效性

### 9.3 稳健性检验
#### 参数敏感性
- 检查参数微调对策略表现的影响
- 确保策略对参数变化不敏感

#### 市场适应性
- 在不同市场环境下验证策略表现
- 确保策略的普适性

### 9.4 实盘模拟
#### 纸面交易
- 在实盘环境中验证策略逻辑
- 检验系统稳定性

#### 微量实盘
- 小资金实盘验证
- 验证执行成本和滑点影响
