# 离线K线缩放后无法缩小的视窗域修复 (XTR-016)

## Intent
在离线查看器中，K 线经过多次放大（zoom in）后出现“无法再缩小（zoom out）”的问题，导致用户无法返回更大时间窗口。该问题直接影响核心分析交互，属于高优先级可用性缺陷。  
本任务目标是在保留当前“按可视窗口动态重采样（LOD）”能力的前提下，修复缩放可逆性，确保用户可随时从局部放大状态退回全局视图。

## Requirement
- 功能目标
  - 修复 K 线图在连续放大后无法缩小的问题，保证 zoom in/zoom out 双向可用。
  - 保留现有视窗级 LOD 重采样逻辑（raw/agg 自动切换），不回退为静态全量渲染。
  - K 线刷新时不得让可缩放数据域被“当前局部子集”持续收缩。
  - 交易标注与 Y 轴动态范围仍需与当前可视窗口联动刷新。
- 非目标 / 范围外
  - 不变更回测产物结构与 run_manifest schema。
  - 不调整 Timeline/Trade Focus 的业务语义。
  - 不新增后端服务或在线依赖。
- 输入输出或接口
  - 输入：离线查看器中已加载的标准 run 目录（含 `run_manifest.json` 与 parquet 数据）。
  - 输出：任意缩放序列下，K 线均可继续放大与缩小；用户可回到更大时间范围（直至全局）。

## Design
- 核心思路与架构
  - 将 K 线 x 轴可缩放域固定为当前窗口数据源（`state._klineSource`）的全局时间边界（`min_ts ~ max_ts`）。
  - 保留“当前可视窗口内重采样并更新 series 数据”的机制，但避免 series 子集反向影响 x 轴全域边界。
  - dataZoom 事件触发时依旧执行：
    1. 计算可视范围
    2. 对可视范围数据做 LOD 重采样
    3. 刷新 K 线数据、交易标注、Y 轴
- 数据/接口/模型
  - 在 K 线 setOption 中显式设置 `xAxis.min` / `xAxis.max`（取 `_klineSource` 首尾时间），并在后续刷新中保持稳定。
  - 修正 `visible.length === 0` 边界处理：不回退到全量绘制导致跳变，改为保留最近有效可视结果或最小邻域兜底。
  - 保持 `refreshKlineViewport()` 的 `requestAnimationFrame` 节流策略。
- 风险与权衡
  - 风险：固定 x 轴域后，某些极端窗口下可能出现短暂空白区。
  - 权衡：优先保证缩放可逆与可预期；空白区通过邻域兜底或状态提示优化。
  - 风险：局部重采样与 dataZoom 交替更新可能带来轻微跳帧。
  - 权衡：维持增量 setOption + raf 节流，优先保证交互流畅。

## Acceptance
- 可验证的验收标准（列举具体检查或指标）
  - 从全局视图连续放大到局部后，用户可通过滚轮/拖拽/slider 继续缩小回更大时间范围。
  - 不出现“放大到某一层级后无法继续 zoom out”的锁死状态。
  - 缩放过程中 K 线仍按当前可视窗口动态切换 raw/agg 粒度。
  - 交易标注与 Y 轴范围在缩放过程中保持联动更新，不出现明显回退。
  - 自动化回归测试通过，且不破坏离线加载与现有交互链路。
