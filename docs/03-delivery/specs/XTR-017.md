# 离线报告UI支持多K线主源手动切换 (XTR-017)

## Intent
当前离线报告 UI 的主 K 线数据源基本固定为 `prices`（通常对应 `5m`），当策略同时依赖 `D/4H/1H/15m/5m` 等多周期决策时，用户无法在同一 Run 内直接切换主 K 线观察口径。  
本任务目标是为离线 UI 引入“多 K 线主源切换”控件：支持手动选择主 K 线周期，并提供默认 `Auto` 模式，且 `Auto` 以“执行周期优先”。

## Requirement
- 功能目标
  - 在 K 线区域增加“主K线源”控件，支持：`Auto`、`5m`、`15m`、`1h`、`4h`、`1d`（按 Run 可用数据动态显示/启用）。
  - 默认模式为 `Auto`。
  - `Auto` 选择规则必须体现“执行周期优先”：若执行周期数据源可用，优先作为主 K 线源；不可用时按“更邻近周期优先”回退，而非固定列表回退。
  - 用户手动选择某周期后进入 `Manual` 状态，不被 Auto 逻辑覆盖；只有用户显式切回 `Auto` 才恢复自动策略。
  - 切换主 K 线源时，保持当前时间窗口（dataZoom）不跳变，并同步刷新 K 线序列、Y 轴、交易标注。
  - v1 的多周期主源读取采用 `snapshot 懒加载`：首次切换到某周期时加载对应快照文件并缓存，后续切换复用缓存。
  - 现有 K 线视窗内桶聚合（OHLC）与缩放联动能力需继续生效，不因主源切换退化。
  - 控件需明确展示当前状态（`Auto/Manual`）与当前主源周期，避免误读。
- 非目标 / 范围外
  - 本任务不调整策略回测计算口径（仅影响 UI 展示主源与读取策略）。
  - 本任务不强制引入跨周期“自动随缩放切源”复杂策略（例如 1d->4h->1h->15m->5m 自动阶梯切换）。
  - 本任务不改动 Trade Ledger、Timeline、Equity 的业务语义。
  - v1 不实现“多周期 chunk 化读取”（该能力放入后续版本）。
- 输入输出或接口
  - 输入：标准 run 目录（manifest + snapshots/chunks），且可包含多周期价格快照（基础 + resampled）。
  - 输出：用户可在 UI 中切换主 K 线源周期；默认 Auto 且执行周期优先；切换后图表联动与交互保持一致。
  - 兼容：若旧 run 未提供多周期主源元信息，UI 应回退到现有单主源逻辑并给出可理解提示。

## Design
- 核心思路与架构
  - 采用“主源选择层 + 现有渲染层”解耦：新增主源选择状态机，不重写既有 K 线渲染与 LOD 逻辑。
  - UI 状态新增：
    - `klineSourceMode`: `auto | manual`
    - `activeTimeframe`: 当前主源周期（如 `15m`）
    - `availableTimeframes`: 当前 run 可用周期集合
  - 控件交互：
    - 初始进入 `Auto`，由 Auto 规则计算 `activeTimeframe`
    - 手动点周期 -> `manual`
    - 点 `Auto` -> 重新按自动规则计算
- 数据/接口/模型
  - Manifest 扩展建议（向后兼容）：
    - 新增 `kline_sources`（可选）描述可用周期、对应 artifact/chunk 路径、默认执行周期信息。
    - 若不存在 `kline_sources`，则 UI 从 `base_timeframe + snapshots/resampled/resampled_manifest.json` 推断可用周期。
  - Auto 规则（v1）：
    - 第一步：从 manifest 推断执行周期（优先 `interval_ms` 对应周期，如 `900000 -> 15m`）。
    - 第二步：若该周期可用，则设为 `activeTimeframe`。
    - 第三步：若不可用，按“邻近周期优先”选择：在可用周期集合中，按与执行周期 `interval_ms` 的绝对差值从小到大排序后取第一项。
    - 第四步（等距决策）：若两侧周期与执行周期距离相同，优先更细粒度周期（更小 `interval_ms`）；若存在同周期重复候选，则优先 `base_timeframe` 对应的数据源。
  - 数据读取：
    - `activeTimeframe == base_timeframe` 时，优先复用现有 `prices` chunk/full parquet 链路。
    - v1 中 `activeTimeframe != base_timeframe` 时，读取 `snapshots/resampled/price_<tf>.parquet` 并缓存到内存（snapshot 懒加载）。
    - 多周期 chunk 化读取作为 v2 演进项，不纳入本次交付范围。
  - 切换行为：
    - 保留当前 dataZoom 范围（start/endValue）并在新序列上重绘。
    - 交易标注按当前可视窗与当前主源坐标刷新。
- 风险与权衡
  - 风险：多周期源切换后，用户可能误解“展示周期”与“执行周期”关系。
  - 权衡：在控件与状态栏明确显示“当前主源周期 + 模式 + 执行周期”。
  - 风险：非基础周期无 chunk 时，大文件一次性加载带来首切换延迟。
  - 权衡：采用懒加载 + 缓存，首切换可接受，二次切换快速。
  - 风险：旧 run 元数据不完整导致可用周期识别不稳定。
  - 权衡：保留旧逻辑兜底，并在状态栏提示“已降级为单主源模式”。

## Acceptance
- 可验证的验收标准（列举具体检查或指标）
  - K 线区出现“主K线源”控件，包含 `Auto` 与 run 内可用周期选项。
  - 默认加载为 `Auto` 模式，且当执行周期数据可用时，主源周期等于执行周期。
  - 当执行周期缺失时，Auto 能回退到“时间跨度更邻近”的可用周期，而不是固定优先级列表。
  - 当出现等距候选周期时，Auto 选择更细粒度周期；若存在同周期重复候选，则优先 `base_timeframe` 对应数据源。
  - 手动选择周期后切换为 `Manual`，再次缩放/拖动不会自动改回其他周期。
  - 用户切回 `Auto` 后，主源重新按“执行周期优先”规则计算。
  - 切换主源时，时间窗口保持不跳变，K 线/Y轴/交易标注同步刷新。
  - 首次切换到非基础周期时可完成该周期 snapshot 加载并渲染；再次切换回该周期时应命中缓存，不重复全量解析文件。
  - 在“仅 base + 15m”run 中，可稳定在 `5m` 与 `15m` 间切换。
  - 在旧 run（无 `kline_sources`）中不崩溃，能回退到现有单主源链路。
  - 现有缩放、LOD 聚合、账本联动、Timeline/Equity 交互不回归。
