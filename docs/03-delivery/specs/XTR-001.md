# Bitget exchange interaction core (XTR-001)

## Intent
- 为 xtrader 项目实现 Bitget 交易所交互的基础能力，使研究和策略模块能够从统一接口获取账户、持仓以及多市场行情。  
- 当前仅有 `ExchangeClient` 抽象和核心模型，缺乏真实交易所实现，导致后续策略无法接入真实数据。

## Requirement
- **功能目标**
  - 提供 `BitgetClient`（命名待定）实现，覆盖：
    1. 账户余额查询（spot、USDT 本位、币本位等账户类型）。
    2. 持仓信息查询（线性/反向合约）。
    3. 历史 K 线拉取（spot、linear_swap、inverse_swap）。
    4. API 配置项（REST base url、api key/secret/passphrase、产品线标识等），并提供可运行的连接性验证（integration test）来确保 URL 与认证配置正确。
    5. 允许通过配置（或 `.env`）注入 HTTP/HTTPS 代理地址，以便在受限网络中访问 Bitget。
  - 支持根据 `MarketType` 自动映射 Bitget 的 `productType`/`symbol`.
  - 提供简易节流/重试机制，保证最小可用。
- **非目标 / 范围外**
  - 下单/撤单等交易执行。
  - 资金划转、子账户管理等高阶账户功能。
  - 利用私有 WebSocket 通道，每日定时任务等。
- **输入输出或接口**
  - 输入：配置对象（API key/secret/passphrase、市场偏好）、方法参数（symbols、interval、时间范围等）。
  - 输出：复用 `xtrader.common.models` 中的数据结构。

## Design
- **核心思路与架构**
  - 在 `src/xtrader/exchanges/bitget.py` 中继承 `ExchangeClient`。
  - 组合 REST 客户端（基于 `httpx`，同步实现）与 WebSocket 客户端（`websockets` 或 `websocket-client`，异步）。
  - 引入配置数据类（`BitgetConfig`）统一处理 host、api key、market mapping。
- **数据/接口/模型**
  - 模型沿用 `AccountBalance`、`Position`、`Candle` 等，内部解析 Bitget JSON。
  - `MarketMeta` 由静态列表 + 在线查询组合。
  - 保存 `symbol_map`，支持现货与合约（`BTCUSDT` -> `BTCUSDT_UMCBL` 等）。
- **风险与权衡**
- 速率限制：先提供基础 sleep + 重试，后续再接入更复杂的 limiter。
- 网络/认证错误：统一包装为自定义异常，便于上层捕获。
- 实时流：本阶段不实现 WebSocket K 线，后续独立任务补充。

## Acceptance
- 运行单元/契约测试时：
  1. 模拟 REST 响应，验证余额/持仓/历史 K 线解析正确。
  2. `BitgetClient.supports()` 返回账户、仓位、历史 K 线功能（不含实时）。
  3. 可选：当环境提供真实 API 凭证时，运行集成测试，确认能与 Bitget API 通讯并返回数据。
- 文档：
  - 在 Spec 与 README 中描述如何配置 Bitget API。
- 实际调用：
  - 使用演示脚本或 notebook 说明如何初始化客户端并拉取示例数据（可用 mock 或最少真实调用）。

## Notes
- 依赖：`httpx`, `pydantic`/dataclasses（现有依赖需检查），可能新增 `websockets`。
- 配置：API key/secret 不纳入仓库，使用环境变量或 YAML。
- 安全：签名算法采用 Bitget 文档（timestamp+method+path+body）。
