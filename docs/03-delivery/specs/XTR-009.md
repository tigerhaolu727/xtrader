# 因子评估流水线与报告标准化 (XTR-009)

## Intent
当前系统已完成因子计算（XTR-007）与因子产物存储（XTR-008），但缺少统一评估流水线，导致 IC、分层收益、稳定性等指标输出口径不一致，难以形成可复现的筛因标准。本任务目标是建设标准化评估流程与 CLI，支持从 `FeatureStore` 一键产出评估结果与报告，作为后续策略研发准入门槛。

## Requirement
- 功能目标
  - 提供统一评估入口：基于 `FeatureStore` identity 读取因子产物并关联价格数据计算标签。
  - 输出统一指标集：
    - IC/ICIR（横截面 Spearman 口径）
    - 分层收益（Top/Bottom/Spread、胜率、Sharpe）
    - 稳定性（正 IC 占比、符号翻转率、IC 一阶自相关、有效覆盖率）
  - 支持参数化评估：
    - `horizon_bars`
    - `execution_lag_bars`
    - `quantiles`
    - 可选时间窗口与 symbol 过滤
  - 提供 CLI 评估命令，输出结构化 JSON 并落盘 Markdown 报告。
  - 评估结果需可复现，报告中必须包含 identity、参数、时间窗、样本量。
- 非目标 / 范围外
  - 本任务不实现组合构建、资金管理、交易成本建模与完整策略回测。
  - 本任务不引入在线评估服务与数据库。
  - 本任务不实现多因子自动搜索/自动调参。
- 输入输出或接口
  - 输入：
    - `FeatureIdentity`（`exchange/market_type/symbol_scope/interval/factor_id/factor_version/params_hash/data_version`）
    - 本地 Kline 数据（用于 forward label）
    - 评估参数与时间窗
  - 输出：
    - 评估摘要对象（含 IC、分层、稳定性）
    - 结构化 JSON（stdout）
    - Markdown 报告（`reports/features/XTR-009/<run_id>/evaluation_report.md`）

## Design
- 核心思路与架构
  - 在 `FactorEvaluator` 上扩展统一评估聚合接口 `evaluate_from_store(...)`，内部复用 `compute_ic_from_store` 并新增分层收益与稳定性计算。
  - 使用统一数据对齐流程：`features` 与 `forward labels` 按 `timestamp+symbol` 内连接后评估。
  - 新增 CLI `python -m xtrader.factors.evaluate_factors`，负责参数解析、读取本地 Kline、调用评估器、落盘报告。
- 数据/接口/模型
  - 新增评估结果对象：
    - `FactorLayeredSummary`
    - `FactorStabilitySummary`
    - `FactorEvaluationSummary`
  - 关键接口：
    - `FactorEvaluator.compute_layered_returns_from_store(...)`
    - `FactorEvaluator.compute_stability_from_store(...)`
    - `FactorEvaluator.evaluate_from_store(...)`
  - CLI 核心参数：
    - identity 参数：`--exchange --market-type --symbol-scope --interval --factor-id --factor-version --params-hash --data-version`
    - 评估参数：`--horizon-bars --execution-lag-bars --quantiles`
    - 数据参数：`--symbols --start-time --end-time --data-root --report-root`
- 风险与权衡
  - 风险：多 symbol 数量不足导致分层收益无效。
    - 对策：分层计算按窗口最小可分层条件过滤，不满足时返回 0 样本并明确标记。
  - 风险：时间对齐错误导致指标失真。
    - 对策：统一 UTC 与 `[start, end)` 约束，评估前做 merge 后样本量检查。
  - 风险：不同场景重复实现评估逻辑造成口径漂移。
    - 对策：统一通过 `FactorEvaluator` 输出，CLI 仅做编排。

## Acceptance
- 可验证的验收标准（列举具体检查或指标）
  - 可通过代码接口输出 IC、分层收益、稳定性三类指标，且包含样本量字段。
  - CLI 可执行并输出结构化 JSON，且同时生成 Markdown 报告文件。
  - 评估支持窗口参数调整（`horizon/execution_lag/quantiles`），参数非法时报错。
  - 单元测试覆盖：
    - 评估结果非空的正向样例
    - 分层样本不足时的稳定返回
    - CLI 运行与报告落盘
  - 回归测试通过，`task_guard.py check XTR-009` 通过。
