# 回测报告UI支持Parquet并移除CSV输出 (XTR-014)

## Intent
当前回测产物会同时输出 Parquet 与 CSV，但离线报告 UI 实际只消费 CSV，造成重复落盘与额外缓存负担。  
本任务目标是将报告 UI 数据读取链路统一到 Parquet，并同步移除回测中的 CSV 输出，以减少报告体积、降低 I/O 成本，并统一数据消费格式。

## Requirement
- 功能目标
  - 报告 UI 必须支持直接读取 Parquet 数据文件，不再依赖 CSV 解析链路。
  - 回测输出中移除 CSV 产物，包括但不限于：`trades.csv`、`equity_curve.csv`、`timelines/signal_execution.csv`、`snapshots/base/price_5m.csv`、`snapshots/action_input.csv`。
  - `run_manifest.json` 与相关索引字段需指向 Parquet 路径，确保 UI 可仅通过 manifest 完成数据发现与加载。
  - UI 在 Parquet 缺失或读取失败时需提供清晰错误提示，便于定位问题。
  - UI 既有展示逻辑与交互能力不得因格式切换退化，至少覆盖：`5m/15m/1h/4h/1d` 多时间粒度展示（由基础/重采样数据提供）、窗口内 OHLC 桶聚合兜底、交易记录标注与联动、Timeline 聚合/明细双模式。
- 非目标 / 范围外
  - 本任务不要求兼容读取旧 CSV-only 报告目录。
  - 本任务不新增后端服务能力（保持离线模式）。
  - 本任务不调整回测指标计算口径与策略逻辑。
- 输入输出或接口
  - 输入：标准回测 run 目录与 `run_manifest.json`。
  - 输出：UI 可加载并展示 Parquet 数据驱动的 K线、交易记录、资金曲线、Signal/Execution Timeline。
  - 数据契约：manifest 中核心表类 artifact 的 `path` 仅指向 `.parquet` 文件。

## Design
- 核心思路与架构
  - UI 侧：用 Parquet 读取器替换现有 CSV 解析路径，统一数据加载入口为 manifest + parquet artifacts。
  - 回测侧：保留现有 Parquet 产物；删除并清理对应 CSV 写出逻辑与 CSV 相关索引字段。
  - 兼容策略：本任务按“新产物规范”执行，不保留 CSV fallback。
  - 实现边界：优先改造数据读取与字段映射层，尽量不改动图表渲染与联动逻辑，以降低功能回归风险。
- 数据/接口/模型
  - `run_manifest.json` 中表类 artifact 保持 `type: table`，`path` 切换为 Parquet 文件相对路径。
  - chunk 化数据（如适用）保持既有时间窗索引能力，但底层数据文件格式统一为 Parquet。
  - 报表入口/索引（如 hub 索引）中交易与权益路径字段统一切换为 Parquet。
- 风险与权衡
  - 风险：移除 CSV 后，旧 UI 版本无法读取新 run。
  - 权衡：通过同步升级 UI 与产物协议，换取目录体积和链路复杂度下降。
  - 风险：Parquet 读取链路在浏览器端的性能/兼容性可能出现波动。
  - 权衡：以单一格式减少双格式维护成本，并通过验证覆盖核心数据集与交互场景。

## Acceptance
- 可验证的验收标准（列举具体检查或指标）
  - 新生成 run 目录中不存在上述 CSV 文件。
  - `run_manifest.json` 与相关索引路径均引用 Parquet 文件，不含 CSV 路径依赖。
  - 离线报告 UI 可在无 CSV 的 run 目录中完整展示 K线、交易、资金曲线、Timeline。
  - 与改造前基线 run 对比，关键统计（trade_count、net_return、max_drawdown、win_rate、profit_factor）一致。
  - 对缺失/损坏 Parquet 场景，UI 提示明确且可定位到具体 artifact。
  - 与改造前基线对比，以下行为保持一致：K线多时间粒度切换与窗口内 OHLC 桶聚合、交易 entry/exit 标注与账本点击联动缩放、Timeline 在大窗聚合/小窗明细切换。
