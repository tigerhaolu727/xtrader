# BTC 日内时序策略与事件驱动交易框架 (XTR-011)

## Intent
目标已从“截面因子选币组合策略”切换为“BTC 单标的日内合约交易策略”。  
新任务聚焦：基于 BTC 日内时序信号，实现自动入场、止盈、止损和仓位管理，并通过事件驱动回测与仿真实盘流程验证策略有效性。

## Requirement
- 功能目标
  - 定义统一时序策略接口，输入 BTC 行情/特征与账户状态，输出交易动作而非目标权重。
  - 支持最小动作集合：
    - `ENTER_LONG`
    - `ENTER_SHORT`
    - `EXIT`
    - `HOLD`
    - `REVERSE`
  - 支持日内交易核心参数：
    - `entry_threshold`
    - `stop_loss`（固定或 ATR）
    - `take_profit`
    - `time_stop_bars`
    - `risk_per_trade`
  - 支持策略状态机：
    - `FLAT` / `LONG` / `SHORT`
    - 入场与出场条件互斥校验
  - 支持事件驱动回测（逐笔/逐bar）：
    - 开平仓交易记录
    - 手续费、滑点、资金费率
    - TP/SL 触发日志
  - 支持输出交易级报告：
    - 交易笔数、胜率、盈亏比、Profit Factor、Expectancy、最大回撤、fee 后净收益
    - 可视化复盘报告（K 线 + 入场/出场标注 + 信号/执行时间轴 + 单笔交易筛选回放 + 前后 N 根窗口聚焦 + 交易详情卡片）
- Phase B（当前阶段）MVP 范围
  - 回测口径：单标的 BTC 日内事件驱动闭环，不使用组合收益近似。
  - 执行模型：先落地 `taker-only`，并预留 `maker` 扩展点。
  - 撮合时点：统一采用“信号产生后下一根 bar 执行”（防止未来函数）。
  - 成本默认值：
    - 手续费：`maker=2bps`, `taker=6bps`
    - 滑点：bps 参数化（可配置）
    - 资金费率：按持仓与时间点计入损益
  - 输出产物：
    - `action events`
    - `execution reports`
    - `trade ledger`
    - `summary metrics`
  - 架构重构约束（主路径收敛）
    - 主路径仅保留 BTC 单标的自动交易闭环：`feature -> action -> state_machine -> risk -> execution -> ledger -> report`。
    - 移除截面选币/组合构权相关代码与测试，不再保留 legacy 兼容入口。
    - 移除历史因子研发子系统（`xtrader.factors`）与 `FeatureStore`，不做旧因子到 CTA 主链路的适配。
    - 新增功能默认以单标的事件驱动接口开发，多标的扩展作为后续独立任务。
- Phase D（长周期可视化报告优化，local static server）
  - 背景：3 年 5m 回测（30 万+ bars，6 万+ trades）下，单文件内嵌全量交易导致 HTML 体积过大、浏览器卡顿。
  - 目标：在不牺牲复盘细节的前提下，实现“秒开 + 可交互 + 自动加载”的长周期报告体验。
  - R1（对应 1）：Trade Ledger 分页/虚拟列表化
    - 默认只渲染当前页（例如 100 行），支持分页与排序。
    - 单笔高亮/筛选逻辑保持不变，且与图表联动。
  - R2（对应 2）：外部数据分片 + 自动按需加载
    - 报告输出改为 `dashboard.html + manifest.json + chunks/*.json` 结构。
    - 前端根据“当前时间范围/视窗”自动加载对应分片，不需要用户手动加载。
    - `file://` 打开时给出提示与一键本地静态服务器启动说明；在 `http://localhost` 模式下启用自动加载。
  - R3（对应 3）：时间范围控制与跨面板联动
    - 支持预设时间窗：`7D/30D/90D/1Y/ALL` 与自定义起止时间。
    - K 线、时间轴、权益曲线、Trade Ledger 使用统一时间过滤条件。
  - R4（对应 4）：长周期视图优化
    - K 线与时间轴采样阈值参数化（不再硬编码 `400/500`）。
    - 增加聚合视图（如 `1h/4h/1d`）与“单笔交易放大视图”（trade spotlight）。
    - 在长周期模式下默认展示聚合视图，进入单笔回放时自动切换到细粒度窗口。
- 非目标 / 范围外
  - 本任务不做多标的截面选币与组合优化。
  - 本任务不直接上线真实实盘下单（先事件回测 + paper trading）。
  - 本任务不实现分布式撮合仿真或高频订单簿级回测。
  - 本任务不继续在主流程增加新的截面因子排序与分位分组能力。
  - 本任务不保留原有因子评估/存储链路（`FactorEngine/FactorEvaluator/FeatureStore`）。
- 输入输出或接口
  - 输入：
    - BTC 日内数据（至少 1m/5m K 线，建议扩展 mark price、funding、bid/ask）
    - 策略信号输入与策略参数（由策略模块直接消费，不依赖旧因子仓库）
    - 当前仓位/订单状态
  - 输出：
    - 策略动作流（action events）
    - 订单与成交流水
    - 回测指标与交易明细报告
    - 成本归因（fee/slippage/funding）与净收益拆解

## Design
- 核心思路与架构
  - 目标流水线：  
    `Market Events -> Signal Builder -> Action Strategy -> Position State Machine -> Risk Manager -> Execution Simulator -> Trade Ledger`
- 模块建议：
    - `src/xtrader/strategies/base.py`：从 `target_weight` 协议扩展为动作型策略协议（action + risk params）
    - `src/xtrader/strategies/state_machine.py`：仓位状态转移
    - `src/xtrader/strategies/risk.py`：止盈止损/单笔风险/日内风控
    - `src/xtrader/execution/`：订单模型与执行仿真器
    - `src/xtrader/backtests/`：新增事件驱动回测器（不再仅依赖组合收益近似）
    - `src/xtrader/backtests/report_bundle.py`（建议新增）：负责 `manifest + chunks` 报告打包与索引构建。
- 数据/接口/模型
  - 关键对象建议：
    - `SignalEvent`
    - `OrderIntent`
    - `ExecutionReport`
    - `PositionState`
    - `TradeRecord`
    - `BacktestSummary`
  - 策略身份建议：
    - `strategy_id + strategy_version + params_hash + data_version`
  - 交易日志最小字段：
    - `timestamp, side, action, qty, entry_price, exit_price, fee, pnl, exit_reason`
  - 资金与成本归因最小字段：
    - `gross_pnl, fee_cost, slippage_cost, funding_cost, net_pnl`
  - 报告分片清单（Phase D 建议）：
    - `report_manifest.json`：时间范围、分片索引、默认视窗、采样参数。
    - `chunks/trades_<yyyy_mm>.json`：交易明细分片。
    - `chunks/actions_<yyyy_mm>.json`：信号/执行事件分片。
    - `chunks/equity_<yyyy_mm>.json`：权益曲线分片。
- 风险与权衡
  - 风险：策略信号有效但执行成本吞噬收益。
    - 对策：把滑点/资金费率/手续费纳入默认口径，禁止仅看毛收益。
  - 风险：状态机与回测器口径不一致导致研究-上线偏差。
    - 对策：策略动作接口在回测与未来实时执行中复用同一实现。
  - 风险：过早引入复杂执行细节拖慢迭代。
    - 对策：先落地 taker-only 简化执行，再逐步扩展 maker 路径。

## Acceptance
- 可验证的验收标准（列举具体检查或指标）
  - 动作型策略接口可运行，输出动作流并通过 schema 校验。
  - 状态机可正确处理 `ENTER/EXIT/REVERSE`，并记录退出原因（止盈/止损/时间止损/信号反转）。
  - 事件驱动回测器可生成交易明细和汇总指标，且结果可复现。
  - 报告包含 fee 后指标：
    - `net_return`
    - `max_drawdown`
    - `profit_factor`
    - `expectancy`
    - `win_rate`
  - 回测可复现：相同输入/参数下重复运行，交易笔数与关键指标一致。
  - 可视化报告可用：`event_driven_report.html` 至少包含
    - `Kline + Trades`
    - `Signal vs Execution Timeline`
    - `Trade Ledger` 与单笔筛选器（`Replay Trade`）。
    - K 线窗口聚焦控制（`Window Bars`）。
    - 交易详情卡片（支持按筛选与 hover 查看）。
  - 长周期报告（3 年 5m）可用性：
    - 采用分片模式后，主 `dashboard.html` 不内嵌全量交易行。
    - 时间范围切换触发自动加载对应分片，并更新图表与表格。
    - 默认视窗下页面可正常交互（筛选、hover、分页、窗口切换不卡死）。
  - 本地静态服务器模式：
    - 文档提供标准启动方式（例如 `python -m http.server`），并在 `file://` 场景给出明确提示。
  - 账本可对账：`gross_pnl - fee_cost - slippage_cost - funding_cost = net_pnl`。
  - 退出原因完整：每笔平仓均有 `exit_reason`（如 `signal/stop_loss/take_profit/time_stop/reverse/daily_loss_limit`）。
  - `python scripts/task_guard.py check XTR-011` 通过。
