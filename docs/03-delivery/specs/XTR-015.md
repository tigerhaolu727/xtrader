# 离线报告UI按可视窗口动态K线粒度 (XTR-015)

## Intent
当前离线报告 UI 在 `ALL` 窗口下会先做 K 线聚合，但用户后续放大时间窗口时，K 线仍停留在初始聚合结果，无法随可视范围动态细化。  
本任务目标是引入“视窗驱动 LOD（Level of Detail）”机制：大窗口显示聚合 K 线，缩放放大后自动重算并细化，最小可回到原始 `5m` K 线，以兼顾性能与可读性。

## Requirement
- 功能目标
  - 数据加载入口仅保留“Run 目录”选择，不再提供独立 `run_manifest.json` 文件选择控件，避免目录与 manifest 错配。
  - 选择 Run 目录后，系统自动在目录内发现并读取 `run_manifest.json`，并据此加载同目录树下的数据文件。
  - K 线展示以原始 `5m` 数据为真值源，不改变回测产物口径。
  - 布局上需将 `Equity Curve` 放置在 `Kline` 正下方，形成“价格-资金曲线”连续对比视图。
  - 页面主体采用全纵向布局：`Kline -> Equity -> Signal vs Execution Timeline -> Trade Ledger`。
  - `Signal vs Execution Timeline` 默认提供全局模式（Global），并支持按单笔交易聚焦模式（Trade Focus）。
  - 点击 `Trade Ledger` 行时，在 Kline/Equity 联动定位的同时，`Timeline` 需切换到该笔交易的聚焦展示。
  - 页面文案以中文为主；关键模块标题可采用“中文（English）”双语样式（如 `资金曲线（Equity Curve）`），兼顾整齐性与专业识别。
  - 图内缩放/拖动（含 scrollbar）作为主时间窗口交互方式，用户可在图上任意放大或缩小。
  - 窗口选择器（如 `7D/30D/90D/365D/ALL`）保留为快捷入口，不作为唯一窗口控制方式。
  - 在大时间范围（如 `ALL`）下可采用聚合 OHLC 展示，以控制渲染负载。
  - 每次图表缩放/拖动（`dataZoom`）后，必须基于“当前可视窗口”重新计算 K 线显示粒度，而非沿用首次渲染结果。
  - 当当前可视窗口内 `5m` K 线数量小于等于像素预算阈值时，必须自动切换为原始 `5m` 显示。
  - 当当前可视窗口内 `5m` K 线数量超过像素预算阈值时，必须自动采用桶聚合 OHLC 显示。
  - 交易标注（entry/exit）需与当前可视窗口同步刷新，不得固定绑定初始窗口或固定时间片。
  - K 线 Y 轴动态缩放需基于“当前实际展示序列（raw/agg）”重算。
- 非目标 / 范围外
  - 不改动回测输出文件结构、manifest schema 与数据字段定义。
  - 不新增本地服务或后端能力，保持离线查看器模式。
  - 不调整 Trade Ledger 与 Timeline 的业务语义与统计口径。
  - 本阶段不实现多语言切换，仅提供 `zh-CN` 默认文案。
- 输入输出或接口
  - 输入：标准 run 目录（含 `run_manifest.json`、价格/交易 parquet chunk 或全量 parquet）。
  - 输出：K 线在缩放过程中可在“聚合/细粒度/原始 5m”间自动切换，且交易标注/Y 轴与可视窗口一致。
  - 事件接口：依赖现有 ECharts `dataZoom` 事件，触发视窗级重采样与图层刷新。

## Design
- 核心思路与架构
  - 页面布局采用“主图纵向分析流”：`Kline -> Equity -> Timeline -> Trade Ledger`，避免左右分栏造成信息割裂。
  - 数据加载采用“目录绑定”模式：用户仅选择 Run 目录，manifest 与 artifacts 需来自同一目录树，消除跨目录手工拼装风险。
  - 采用“视窗优先 + 像素预算”策略：以当前图表可视时间范围为输入，动态计算应展示的 bars 数量。
  - 交互双通道：窗口选择器用于快速定位初始范围；后续以图内 dataZoom/scrollbar 精细控制窗口并触发动态重算。
  - 首屏（尤其 `ALL`）可先聚合；当用户放大到更小时间窗时按需降桶或取消聚合，直到显示原始 `5m`。
  - 缩放刷新时仅更新必要图层（K 线主序列、markPoint、Y 轴），避免整图重建造成抖动。
  - Timeline 双模式：Global 用于整体观察，Trade Focus 用于单笔交易信号执行分析；两种模式可在 UI 中显式切换。
- 数据/接口/模型
  - 基础价格源：`5m` rows。
  - 目录加载规则：Run 目录内必须存在且仅匹配一个 `run_manifest.json`；缺失或冲突时给出明确中文错误提示。
  - 动态阈值：`targetBars ~= chartWidthPx * 2`（可配置常量）。
  - 粒度判断：
    - `visibleRows <= targetBars` -> raw `5m`
    - `visibleRows > targetBars` -> bucket OHLC（`bucket = ceil(visibleRows / targetBars)`）
  - 交易标注：按当前可视窗口过滤后再采样，保持上限控制（避免 marker 过多）。
  - 联动关系：`Trade Ledger` 点击定位时，`Kline` 与 `Equity` 共享时间窗口并同步缩放，`Timeline` 同步切换到该笔交易窗口的数据子集。
  - Trade Focus 的事件过滤策略：优先按交易标识精确关联；若当前数据无交易标识，则按 `entry_time~exit_time`（含 buffer）近似过滤。
  - 文案策略：维护统一中文术语表（如 `K线`、`资金曲线`、`信号与执行时间线`、`交易记录`），关键标题采用“中文（English）”过渡样式，其余文案保持中文优先，避免无规则中英文混杂。
- 风险与权衡
  - 风险：高频缩放触发重算可能导致主线程抖动。
  - 权衡：采用 `requestAnimationFrame` 节流 + 增量 setOption 更新，平衡实时性与性能。
  - 风险：raw/agg 切换边界附近可能出现视觉跳变。
  - 权衡：通过稳定阈值与一致 OHLC 规则降低感知突变。

## Acceptance
- 可验证的验收标准（列举具体检查或指标）
  - 控制区仅提供 Run 目录选择入口；页面不再提供独立 `run_manifest.json` 选择控件。
  - 选择 Run 目录后，系统可自动加载目录内 manifest 与数据；不存在“目录A + manifestB”错配路径。
  - Run 目录缺失 manifest 或 manifest 不唯一时，页面给出明确中文错误提示并阻止加载。
  - 页面布局顺序为：`Kline -> Equity -> Signal vs Execution Timeline -> Trade Ledger`，不再采用 Timeline/Ledger 左右分栏。
  - 页面按钮、状态、表头与提示文案为中文；关键图表标题可采用“中文（English）”双语格式，术语前后一致。
  - 用户浏览时可直观看到同一时间段的价格与资金曲线变化，不需跨大段页面跳转。
  - `ALL` 初始视图下，K 线可为聚合结果，页面加载与交互无明显阻塞。
  - 用户可不依赖窗口选择器，仅通过图内缩放/拖动与 scrollbar 完成窗口选择，并触发粒度动态切换。
  - 窗口选择器仍可正常工作，且与图内缩放行为兼容（选择快捷窗口后可继续细调）。
  - 在 `ALL` 视图连续放大时，K 线粒度会随可视窗口缩小而自动细化（非固定聚合）。
  - 放大到足够小窗口时，K 线显示原始 `5m` 数据（不再聚合）。
  - 再次缩小窗口后，K 线可自动回退为聚合展示。
  - 交易标注在缩放过程中始终与当前可视窗口一致，不出现“长期集中在左侧”问题。
  - `Trade Ledger` 点击缩放时，`Kline` 与 `Equity` 保持同步窗口定位，`Timeline` 自动进入 Trade Focus 并展示该笔交易相关事件。
  - `Timeline` 支持从 Trade Focus 返回 Global，且切换后数据与交互状态一致。
  - Y 轴范围在缩放过程中随当前展示数据动态更新。
  - 相关单测通过，且不回归既有离线加载流程（manifest + parquet）。
