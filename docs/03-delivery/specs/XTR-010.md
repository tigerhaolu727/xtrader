# Walk-forward 策略回测框架 (XTR-010)

## Intent
当前项目已完成数据底座（XTR-006）、因子框架（XTR-007）、因子产物存储（XTR-008）与评估流水线（XTR-009），但缺少样本外验证闭环。单次全样本回测容易高估效果，无法反映参数与因子在时间变化下的稳定性。本任务目标是建设 Walk-forward 回测框架，形成“滚动训练/验证/测试”标准流程，为策略准入提供更可靠依据。

## Requirement
- 功能目标
  - 支持基于时间的滚动窗口切分：`train -> validate -> test` 多折运行。
  - 支持窗口参数化配置：
    - `train_span`
    - `validate_span`
    - `test_span`
    - `step_span`
    - `min_warmup_bars`
  - 支持从 `FeatureStore` 读取因子产物，配合本地 Kline 价格数据完成每折回测。
  - 支持统一策略执行口径（首版）：
    - 横截面排序信号
    - 分位数多空（Top/Bottom）
    - 可配置 `execution_lag_bars`
  - 输出每折结果与总汇总结果：
    - 收益与风险指标（累计收益、年化、波动、Sharpe、最大回撤）
    - 交易行为指标（换手、持仓覆盖）
    - 稳定性指标（各折收益一致性、胜率）
  - 支持生成结构化结果文件与 Markdown 报告。
  - 强制防泄漏约束：窗口无重叠违规、训练集信息不可泄漏到测试折。
- 非目标 / 范围外
  - 本任务不实现真实下单与实时交易。
  - 本任务不实现大规模自动调参（AutoML）或分布式搜索。
  - 本任务不引入交易成本复杂模型（首版仅保留简单成本占位参数）。
  - 本任务不实现组合级风险预算优化（如 ERC、HRP）。
- 输入输出或接口
  - 输入：
    - `FeatureIdentity`（单因子或多因子列表）
    - 策略配置（分位数、执行延迟、成本参数）
    - Walk-forward 配置（窗口跨度与步长）
    - 时间范围与 symbols
  - 输出：
    - 每折明细结果（JSON）
    - 汇总结果（JSON）
    - 回测报告（`reports/backtests/XTR-010/<run_id>/walk_forward_report.md`）

## Design
- 核心思路与架构
  - 新增 `WalkForwardPlanner`：根据总时间范围生成多折时间窗口。
  - 新增 `WalkForwardRunner`：逐折执行数据加载、策略运行、指标计算。
  - 新增 `FoldMetricsAggregator`：聚合各折指标并输出稳定性统计。
  - 新增 CLI 入口：`python -m xtrader.backtests.walk_forward`，负责配置解析与结果落盘。
- 数据/接口/模型
  - 核心对象建议：
    - `WalkForwardConfig`
    - `FoldWindow`（`train_start/end`, `validate_start/end`, `test_start/end`）
    - `FoldResult`
    - `WalkForwardSummary`
  - 关键接口建议：
    - `plan_windows(config) -> list[FoldWindow]`
    - `run_fold(window, strategy_config, identities) -> FoldResult`
    - `run_walk_forward(config) -> WalkForwardSummary`
  - 结果路径建议：
    - `reports/backtests/XTR-010/<run_id>/folds/fold_<n>.json`
    - `reports/backtests/XTR-010/<run_id>/summary.json`
    - `reports/backtests/XTR-010/<run_id>/walk_forward_report.md`
- 风险与权衡
  - 风险：窗口设计不合理导致样本数不足，指标波动过大。
    - 对策：提供最小样本阈值检查，样本不足折标记为 `insufficient`。
  - 风险：时间切分实现错误引入泄漏。
    - 对策：在 planner/running 阶段加入严格时间边界断言与守门测试。
  - 风险：多因子/多标的时运行耗时上升。
    - 对策：首版先保证正确性，后续再做缓存与并行优化。

## Acceptance
- 可验证的验收标准（列举具体检查或指标）
  - 可基于配置成功生成至少 3 个有效折（窗口边界正确、无重叠违规）。
  - 每折可产出独立结果文件，且包含收益、风险与交易行为指标。
  - 汇总结果可输出跨折稳定性统计（均值、标准差、胜率等）。
  - CLI 可运行并输出报告文件路径。
  - 防泄漏校验通过：无使用未来数据、无折间边界越界。
  - 相关单测与回归通过，`task_guard.py check XTR-010` 通过。
