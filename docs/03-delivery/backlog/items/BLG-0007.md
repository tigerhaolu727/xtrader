## Context
- Source discussion:
  - 基于 `docs/02-strategy/discussions/trading-strategy-bg.md` 中 `3.1 / 维度五：波动率分析` 的实现讨论。
  - 文档当前要素为：`ATR百分比`、`布林带宽度`、`历史波动率`。
- Problem or opportunity:
  - 三项特征存在较高同质性，可能重复计分同一类“波动大小”信息。
  - 当前未明确“高波动/低波动”的加减分方向，难以适配不同风格策略。
  - 缺少“状态转移”表达（扩张/收缩切换），解释力不足。
- Why not now:
  - 五维评分总体尚未冻结，先收敛设计口径，暂不编码。

## Desired Outcome
- 冻结维度五 v1 的环境化评分口径，明确：
  - 维度五定位为“市场环境评分”，不直接承担方向触发职责；
  - 与维度三/四的边界清晰（避免重复计分）；
  - 输出统一分数（0-100）与解释标签，支持回测复盘。

## Proposed Direction
- 维度角色收敛：
  - 维度五只评估“当前波动环境是否适合交易/适合哪类触发”，不判定多空方向。
- v1 特征建议：
  - `atr_pct`：`atr / close` 的分位化值（percentile）。
  - `bb_width`：`(bollinger_up - bollinger_low) / bollinger_mid` 的分位化值。
  - `hist_vol`：收益率 rolling std 的分位化值。
  - `vol_regime_shift`：波动扩张/收缩状态（例如宽度斜率或短长窗比值）。
- 去重建议：
  - 主特征保留 2 项（`atr_pct` + `bb_width` 或 `hist_vol`）；
  - 加 1 项状态切换特征，避免三项都在表达同一“绝对大小”。
- 评分语义建议：
  - 统一 `0-100`，`50` 为中性；
  - 根据策略风格配置映射方向（突破型偏好扩张，均值回归偏好收缩）。
- 与维度三/四协同：
  - 作为维度三触发信号的环境权重或 gate；
  - 与维度四趋势状态并行，不重复做方向性判定。

## Trigger Condition
- Start implementation when:
  - 五维评分系统进入实现阶段；
  - 或 signal 引擎联调需要波动环境过滤；
  - 或回测显示触发过多但环境适配不足（噪音交易偏多）。
