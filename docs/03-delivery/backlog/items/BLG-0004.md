  - 讨论重点为：程序如何识别“近期支撑/阻力附近”和“前 N 根是否下跌”，并给出一版固定参数默认值。
- Problem or opportunity:
  - 仅做形态几何识别（如锤子线/吞没/十字星）容易误报，需要上下文过滤（位置 + 趋势）提高可用性。

## Desired Outcome
- 冻结维度二 MVP 的上下文识别规则，形成可实现且可回测的默认参数集：
  - 近期支撑/阻力判定；
  - 近期趋势（前 N 根）判定；
  - 与形态信号的组合方式（硬门槛或打分乘子）。

## Proposed Direction
- 支撑/阻力附近（v1）：
  - `recent_support = low.shift(1).rolling(sr_lookback_bars).min()`
  - `recent_resistance = high.shift(1).rolling(sr_lookback_bars).max()`
  - `tol = max(ATR(atr_period) * near_atr_mult, close * near_pct_tol)`
  - `near_support/near_resistance = distance <= tol`
  - 说明：使用 `shift(1)` 保证无前视。
- 近期下跌趋势（v1）：
  - `downtrend_window_bars` 内满足：
    - 阴线数量 `>= downtrend_min_bearish_bars`
    - 净跌幅 `>= downtrend_min_net_drop`
    - 线性斜率 `<= downtrend_slope_max`（默认 0）
- 推荐默认参数（5m/15m 初始口径）：
  - `sr_lookback_bars = 50`
  - `atr_period = 14`
  - `near_atr_mult = 0.5`
  - `near_pct_tol = 0.002`（0.2%）
  - `downtrend_window_bars = 5`
  - `downtrend_min_bearish_bars = 4`
  - `downtrend_min_net_drop = 0.003`（0.3%）
  - `downtrend_slope_max = 0`
