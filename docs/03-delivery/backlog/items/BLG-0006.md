## Context
- Source discussion:
  - 基于 `docs/02-strategy/discussions/trading-strategy-bg.md` 中 `3.1 / 维度四：趋势强度分析` 的实现讨论。
  - 结论：维度四逻辑方向合理，但需与维度三明确边界，避免重复计分。
- Problem or opportunity:
  - 维度四应回答“趋势是否存在、强度是否够、延续是否健康”，与维度三“触发事件”形成互补。
  - 若维度四使用过多交叉/突破事件，会和维度三冲突并降低解释性。
- Why not now:
  - 五维评分尚未整体冻结，先收敛设计口径，暂不编码。

## Desired Outcome
- 冻结维度四 v1 的状态化评分口径，明确：
  - 只评估趋势状态与持续性，不承担事件触发职责；
  - 输出统一分数（0-100）与解释标签；
  - 可作为维度三信号的门控因子（gate）。

## Proposed Direction
- 模块分层：
  - `feature_engine`: 提供 `ema/ma/atr` 等连续指标。
  - `signal/scoring` 层（待建设）: 计算 `trend_strength_score` 与 `trend_reasons`。
- 维度四 v1 子分项（状态化）：
  - `ema_distance_score`：`(ema_12 - ema_26)` 相对价格或 ATR 的归一化距离。
  - `ema_slope_score`：`ema_12/ema_26` 的斜率一致性与方向稳定性。
  - `price_position_score`：价格相对 `ma_20/ema_26` 的位置与偏离合理性。
  - `trend_health_score`：趋势内回撤深度、回撤恢复速度（健康度）。
- 建议合成权重（v1 默认）：
  - `ema_distance=0.30`, `ema_slope=0.30`, `price_position=0.25`, `trend_health=0.15`
- 时间尺度建议：
  - 维度四使用中窗（约 `20-100 bars`），强调持续性和稳定性。
- 与维度三协同：
  - 维度四不输出“买卖触发”，而是输出状态质量；
  - 作为 `technical_score` 的 gate 或加权因子，抑制逆势触发。

## Trigger Condition
- Start implementation when:
  - 五维评分系统进入实现阶段；
  - 或信号引擎联调阶段需要“趋势状态门控”；
  - 或回测中出现事件信号过多、趋势过滤不足的问题。
