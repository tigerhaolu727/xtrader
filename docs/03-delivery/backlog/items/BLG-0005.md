## Context
- Source discussion:
  - 基于 `docs/02-strategy/discussions/trading-strategy-bg.md` 中 `3.1 / 维度三：技术指标分析` 的实现讨论。
  - 当前 `feature_engine` 已具备 RSI/MACD/MA/Bollinger 等连续指标产出，但尚未有评分引擎层进行规则映射与维度聚合。
  - 新增结论：维度三与维度四存在信息重叠风险，需要先做边界去重。
- Problem or opportunity:
  - 维度三需要把连续指标转化为可解释分数（0-100）并输出原因标签，供总分、信号判定和回测复盘使用。
  - 若将规则写入指标层会造成分层混乱，后续调参成本高。
  - 若维度三继续承载“趋势状态”信息，会和维度四重复计分，导致过度放大同一类证据。
- Why not now:
  - 当前优先完成整体策略分析，评分系统总体口径尚未冻结，暂不进入编码。

## Desired Outcome
- 冻结维度三 v1 的程序化设计，明确：
  - 模块边界（指标层 vs 评分层）；
  - 子分项与加权合成；
  - 与维度四的去重边界（事件触发 vs 状态强度）；
  - 输出字段与解释标签规范；
  - 后续可直接升级为 `XTR-xxx` 进入 spec/validation。

## Proposed Direction
- 模块分层：
  - `feature_engine`: 仅产连续指标值，不包含评分逻辑。
  - `signal/scoring` 层（待建设）: 读取 `model_df`，输出 `technical_score` 与 `technical_reasons`。
- 维度三角色（收敛）：
  - 仅负责“事件与触发”（event-driven），不负责“趋势状态强度”。
  - 典型事件：交叉、突破、超买超卖、背离线索。
- 维度三 v1 子分项（事件化）：
  - `rsi_score`：`rsi_14` 区间触发（超买/超卖）+ 短期方向确认。
  - `macd_score`：`macd_line/signal/hist` 的金叉死叉与柱体同向扩张。
  - `ma_alignment_score`：以“排列切换事件”为主（多头形成/空头形成），弱化持续状态加分。
  - `bb_breakout_score`：上/下轨突破与回归事件。
- 建议合成权重（v1 默认）：
  - `rsi=0.25`, `macd=0.30`, `ma_alignment=0.25`, `bb_breakout=0.20`
  - 可选共振加成：四项中三项同向时加小幅 bonus（上限封顶）。
- 时间尺度建议：
  - 维度三使用短窗（约 `5-20 bars`）保证触发敏捷。
- 与维度四联动建议：
  - `effective_technical_score = technical_score * gate(trend_strength_score)`；
  - 当维度四低于阈值时，维度三触发降权而非直接失效。
- 输出契约（建议）：
  - 数值列：`technical_score`（0-100）
  - 过程列：`technical_rsi_score` / `technical_macd_score` / `technical_ma_alignment_score` / `technical_bb_breakout_score`
  - 解释列：`technical_reasons`（字符串列表或 JSON 字符串）
