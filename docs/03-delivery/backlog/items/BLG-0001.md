# BLG-0001: 2.1 数据预处理能力补齐（异常值/有效性校验/插值）

## Meta
- ID: `BLG-0001`
- Status: `parking`
- Priority: `P1`
- Owner: `tiger+codex`
- Created At: `2026-03-31`
- Last Updated: `2026-03-31`

## Context
- Source discussion:
  - 基于策略文档 `docs/02-strategy/discussions/trading-strategy-bg.md` 的 `2.1 数据预处理` 讨论。
  - 结论：当前框架已具备时间对齐、缺口修复、去重排序等基础能力，但仍缺部分策略文档要求。
- Problem or opportunity:
  - 目前缺失以下能力：
    1. 异常值检测（Z-score / IQR）；
    2. 统一 K 线有效性校验（`high>=low`, `open/close` 位于 `[low, high]`, `volume>=0`）；
    3. 缺失值插值策略（可选开关，不影响默认流程）。
- Why not now:
  - 当前优先级是先完成策略分析与规则讨论，暂不进入实现。

## Desired Outcome
- 在不破坏现有下载/回测链路的前提下，新增可配置的数据预处理模块，支持：
  - 预处理报告可追踪（处理了多少条、为何处理）；
  - 默认兼容旧流程（不开启新开关时行为不变）；
  - 可在策略回测前显式执行并落盘产物。

## Proposed Direction
- Option A:
  - 在 `data` pipeline 新增独立预处理步骤（建议 `src/xtrader/data/pipelines/kline_preprocess.py`），输出清洗后数据与报告。
- Option B:
  - 直接在 `kline_sync`/`kline_resample` 内嵌处理逻辑，减少模块数量但耦合更高。
- Recommended:
  - 采用 Option A（独立步骤），便于启停和审计，并避免污染现有稳定链路。

## Trigger Condition
- Start implementation when:
  - BG 策略规则进入“可实现版本”并准备首轮系统回测；
  - 需要对异常 K 线处理口径做可复现审计；
  - 团队确认是否采用“删除异常值”或“winsorize/插值”作为默认策略。
