# Validation Plan (XTR-018)

## Planned Validation
- [x] `task_guard` 流程校验：执行 `python scripts/task_guard.py check XTR-018`，确认 Spec/Validation 结构合规。
- [x] 接口一致性检查（实现阶段）：各指标族均实现统一接口（`validate_params/compute`）与元信息字段。
- [x] 返回类型检查（实现阶段）：所有指标族 `compute` 统一返回 `DataFrame`，单列指标不返回 `Series`。
- [x] 参数实例化检查（实现阶段）：通过 `indicator_plan` 同时运行 `ema_12` 与 `ema_26`，验证同族多实例输出分离且列名稳定。
- [x] `indicator_plan` 契约检查（实现阶段）：v1 单条配置必须包含 `instance_id + family + params`。
- [x] `indicator_plan` 唯一性检查（实现阶段）：同一计划内 `instance_id` 不可重复。
- [x] `indicator_plan` 去重检查（实现阶段）：同一计划内禁止重复 `family + params` 组合。
- [x] 命名规范检查（实现阶段）：输出列名必须符合 `{indicator}_{参数1_参数2}`，必须含参数，禁止别名。
- [x] 多输出命名检查（实现阶段）：多输出指标采用“参数前缀 + 语义后缀”规则；例如 `MACD(12,26,9)` 产出 `macd_12_26_9_line/signal/hist`。
- [x] 非 MACD 多输出后缀检查（实现阶段）：`bollinger(mid/up/low)`、`kd(k/d/j)`、`dmi(plus_di/minus_di/adx)` 与字典一致。
- [x] 参数顺序检查（实现阶段）：列名参数顺序必须遵循指标族 `param_order`，且不受 `params` 字典键顺序影响。
- [x] 默认参数命名检查（实现阶段）：未显式传参时，默认值参与列名生成（如 `macd_12_26_9`）。
- [x] 浮点字符串化检查（实现阶段）：参数值 `2.0 -> 2`；非整数浮点统一保留两位小数（如 `2.3 -> 2.30`）。
- [x] 浮点列名格式检查（实现阶段）：浮点参数列名保留小数点（如 `bollinger_20_2.30_mid`）。
- [x] 输入不可变检查（实现阶段）：调用指标 `compute` 前后，原始输入 DataFrame内容与列集合不变。
- [x] 时间透传检查（实现阶段）：`compute` 前后 `timestamp` 列值、顺序、行数完全一致。
- [x] 模块交付检查（实现阶段）：`feature_pipeline` 产出 `model_df`，并满足后续评分系统消费所需字段契约（评分实现不在本任务范围）。
- [x] 单标的主键检查（实现阶段）：`model_df.timestamp` 非空、唯一、升序；`symbol` 非主键且列值单一。
- [x] 合并口径检查（实现阶段）：指标合并使用 `timestamp` 左连接，合并后行数等于 `bars_df` 行数。
- [x] 重复时间戳错误检查（实现阶段）：输入 `bars_df` 存在重复 `timestamp` 时，流程应 fail-fast 报错。
- [x] 时间口径错误检查（实现阶段）：`timestamp` 缺失/不可解析/非 UTC 口径时返回 `INPUT_*` 错误码并中止。
- [x] 报错规范检查（实现阶段）：失败路径错误消息遵循 `XTR018::<ERROR_CODE>::<detail>`。
- [x] 错误码覆盖检查（实现阶段）：覆盖 `PLAN_* / PARAM_* / INPUT_* / OUTPUT_COLUMN_CONFLICT` 核心场景（含 `PLAN_DUPLICATE_FAMILY_PARAMS`）。
- [x] 指标单测（实现阶段）：按指标族验证参数、warmup、输出列与关键样例值。
- [x] 黄金样本对齐测试（实现阶段）：固定输入样本与指标计划，断言输出列顺序、首个有效索引及尾部数值快照与 fixture 一致。
- [x] 分层边界检查（实现阶段）：确认事件特征（金叉/死叉/突破等）未进入 `feature_engine`，仅在 `signal_engine` 处理。
